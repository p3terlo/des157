Best Practices for Modals

Apr. 23, 2021

Modal windows are elements that sit on top of an application’s main window. They disable the main window while keeping it visible, usually with a lightbox effect highlighting the modal window and darkening the main window for emphasis, with the modal window as a child in front of it. Users are required to interact with the modal window before returning to the parent window. Also known as overlays or dialogs, modals provide an elegant solution to a classic UI problem; namely simplifying the UI while saving screen real estate. Most often they are used for grabbing user attention, requiring user input, and showing additional information without losing the context of the parent application. Unfortunately, modals have become the modern popup window with users often being trained to automatically dismiss these windows. As a result, good design of modal windows should include the following: give users a clear indication of how to close the modal, include a descriptive title to alert users of the purpose, give the modal an appropriate size and location (no more than 50% of the screen real estate and placed in the upper half of the screen), create focus using a lightbox effect, and finally, have the modal be user initiated rather than an automatic popup.

Best Practices for Form Design

Apr. 14, 2021

In reading the article Best Practices for Form Design by Salim Ansari, I learned that when designing forms, a key concept to keep in mind is that less is more. The designer’s job is to make the form as streamlined as possible so that users aren’t overwhelmed by cognitive overload, which ultimately results in the loss of business and customers. In streamlining the form experience, its essential that we minimize the input fields as much as possible and design the layout in a one-column manner while breaking the form down into multiple chunks to minimize user cognitive load and visual noise. It’s also important to distinguish optional and required fields and explain any input or formatting requirements as well as the uses of sensitive information. In essence, treating them with a certain level of respect. Finally, providing proper form feedback in the form of success and specific error messages is key to letting the user know if they’ve finish the form. Accessibility in terms of contrast, text size, screen readers, and not using color as the only visual means to communicate status is also something to keep in mind.
